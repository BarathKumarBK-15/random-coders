Problem 1: Kiruba Under Scrutiny!

Problem Statement

Kiruba, one of the problem setter and tester of RANDOM CODERS, is brought under scrutiny by Barath, the Admin of RANDOM CODERS due to Kiruba's poor setting and unbalanced difficulty in Chapter - 2 which was not at all suitable for beginners. So, to really test his skills, Barath gave him a task to perform. The tasks states that, given a binary string, interchange the positions of 1's and 0's and convert it into its decimal equivalent. Formally, for a given binary string S, perform the following operations.

* If S[i] is '0' change it to '1'.
* If S[i] is '1' change it to '0'.
* Convert the resultant binary string into its decimal equivalent.

Since, Kiruba was really upset about being scrutinized, he wants you to help him to solve this.

Input Format

* The first line contains a single integer, T representing the no. of test cases.
* The first line of each test case contains a single integer, N representing the length of binary string.
* The second line of each test case contains a binary string, S of length N.

Constraints

* 1 <= t <= 100
* 1 <= n < 32
* 0 <= S[i] <= 1

Output Format

* For each test case, print the decimal equivalent of the given binary string after performing the required operations.

Sample Input 0

3
3
101
5
01101
1
1

Sample Output 0

2
18
0

Explanation 0

* Case 1: By converting all one's to zero's and vice versa, 101 become 010 which is binary equivalent of 2
* Case 2: Similarly 01101 becomes 10010 whose decimal equivalent is 18
* Case 3: Similarly 1 becomes 0 whose decimal value is also 0

Approach

- In the given string, if S[i] == '0', change it to '1'.
- If S[i] == '1', change it to '0'.
- Then convert the resultant binary string to its decimal equivalent.

Code in C++

#include <iostream>

using namespace std;

int power(int base , int pow) {
    if(pow == 0)
        return 1;
    if(pow % 2 == 0)
        return power(base * base , pow / 2);
    else
        return base * power(base * base , (pow - 1) / 2);
}

int main() {
    int t;
    cin >> t;
    while(t--) {
        int n;
        cin >> n;
        string s;
        cin >> s;
        int ans = 0 , mul = s.length() - 1;
        for(int i = s.length() - 1; i >= 0; i--) {
            if(s[i] == '0')
                ans += power(2 , mul - i);
        }
        cout << ans << endl;
    }
    return 0;
}

def btod(b):
    dec=0
    term=1
    while len(b):

        b=b[:-1]
    return dec

Problem 2: Kiruba's Performance Crisis...

Problem Statement

Barath is not happy with Kiruba's performance of problem setting in Chapter - 2, so he told Rachit, Designer and Maintance Head of Random Coders to check on Kiruba's performance record for previous N days and told him to check whether he passes the required performance criteria or not. Now, Rachit comes up with a new strategy to compute Kiruba's performance. He converts the linear data of N days into circular form and considers the performance value only if it is strictly greater than the previous and next day's performance. If this condition is satisfied, the performance value is added to Kiruba's net performance. After this process, if the net performance is lesser than minimum performance required to pass the performance criteria print 'NO', else print 'YES'.

Note: Circular data is same as linear data except the fact that, in circular data, the last element is left to first element and the first element is right to last element. Eg: [1,2,3,4], 4 is left to 1 and 1 is right to 4.

Input Format

* The first line contains a single integer, t representing the no. of test cases.
* The first line of each test case contains a single integer, N representing the length of the array.
* The second line of each test case contains N space separated integers, representing circular array, A the performance of Kiruba on each day.
* The third line of each test case contains a single integer, k representing the minimum performance required to pass the performance criteria.

Constraints

* 1 <= t <= 100
* 2 <= N <= 100
* 1 <= A[i] <= 100
* 1 <= k <= 1500

Output Format

* For each test case, print "YES" or "NO" (without quotes).

Sample Input 0

2
5
1 3 4 2 5
8
2
3 3
1

Sample Output 0

YES
NO

Explanation

* Case 1: Kiruba hits high performance on days 2 and 5. Therefore total performance = 4 + 5 = 9, which clears the basic requirement. Thus print "YES"
* Case 2: Similay here the total performance is zero, which is less than the one required. Thus print "NO"

Approach

- If performance of a day is strictly greater than pervious and next day, add it to the net performance.
- If net performance is lesser than the minimum required then print "NO", else print "YES" (without quotes).

Code in C++

#include <iostream>
#include <vector>

using namespace std;

int main() {
    int t;
    cin >> t;
    while(t--) {
        int n;
        cin >> n;
        vector<int> a;
        for(int i = 0; i < n; i++) {
            int element;
            cin >> element;
            a.push_back(element);
        }
        int k;
        cin >> k;
        int ans = 0;
        for(int i = 0; i < n; i++) {
            if(i > 0 && i < n - 1) {
                if(a[i - 1] < a[i] && a[i] > a[i + 1]) {
                    ans += a[i];
                }
            }
            else if(i == 0) {
                if(a[n - 1] < a[i] && a[i] > a[i + 1]) {
                    ans += a[i];
                }
            }
            else {
                if(a[i - 1] < a[i] && a[i] > a[0]) {
                    ans += a[i];
                }
            }
        }
        if(ans < k)
            cout << "NO" << endl;
        else
            cout << "YES" << endl;
    }
    return 0;
}

Problem 3: Kiruba's Mistakes Revealed!

Problem Statement

It was time for Chapter - 3 and the questions set by Kiruba was being tested. Undoubtedly there was a mistake in the question that was set by Kiruba. The original question was, given an array A of N elements, perform the following operations.

* Compute min(A[i] C (A[i] - 1) , A[i] P (A[i] - 1)).
* Print the sum of all such computations.

But in the above question it was found that, A[i] C (A[i] - 1) was always smaller than A[i] P (A[i] - 1). Therefore, it was not at all necessary to take min of those two parameters. So, to escape from being caught, Kiruba modified the question slightly. The new question now states that,

* Compute A[i] C (A[i] - 1), if A[i] is even.
* Compute A[i] P (A[i] - 1), if A[i] is odd.
* Print the sum of all such computations.
* Note 1: Since, the answer maybe very large, print the sum modulo 10^9 + 7.
* Note 2: n C r = (n!)/((n - r)! * r!) and n P r = (n!)/(n-r)!.

Input Format

* The first line contains a single integer, t representing the no. of test cases.
* The first line of each test case contains a single integer, N representing the length of an array.
* The second line of each test case contains N space separated integers, A[i] representing the elements of an array.

Constraints

* 1 <= t <= 10
* 1 <= N <= 10000
* 1 <= A[i] <= 20

Output Format

* Print a single integer, representing the sum of all the required computations modulo 10^9 + 7

Sample Input 0

1
3
2 3 4

Sample Output 0

12

Explanation 0

* The first element 2 is even, combination is 2 C (2-1) which is 2
* Since the second element 3, is odd, it gives 3 P (3-1) which is 6
* Similarly for the last element we get 4
* Thus the sum is 2+6+4=12

Approach

- For the given array, if A[i] is even perform A[i] C (A[i] - 1).
- If A[i] is odd perform A[i] P (A[i] - 1).
- Compute the sum of each operation modulo 10^9 + 7.
- Note: Here A[i] C (A[i] - 1) is equal to A[i] and A[i] P (A[i] - 1) is equal to (A[i])!.

Code in C++

#include <iostream>
#include <vector>
#define mod 1000000007

using namespace std;

int combination(int n , int r) {
    return n;
}

long long permutation(int n , int r) {
    long long fac = 1;
    for(int i = 1; i <= n; i++)
        fac *= i;
    return fac;
}

int main() {
    int t;
    cin >> t;
    while(t--) {
        int n;
        cin >> n;
        vector<int> a(n);
        for(int i = 0; i < n; i++)
            cin >> a[i];
        long long ans = 0;
        for(int i = 0; i < n; i++) {
            if(a[i] % 2 == 0) {
                ans = (ans + combination(a[i] , a[i] - 1)) % mod;
            }
            else {
                ans = (ans + permutation(a[i] , a[i] - 1)) % mod;
            }
        }
        cout << ans << endl;
    }
}

    f=1
    while x:
        f*=x
        x-=1
    return f

Problem 4: Will Kiruba Leave?

Problem Statement

This week was really frustating for Kiruba and his teammates, since there were several rumours that Kiruba is leaving RANDOM CODERS. But surprisingly Kiruba was given a final opportunity to stay, and this was completely based on mere luck. His teammates gave him a string S (with lowercase English alphabets) of length N and told him to perform the following operations any no. of times (possibly zero).

* For valid i, remove S[i] from the string. After performing the given operation, if the resultant string is "kiruba", he will not leave RANDOM CODERS, else he will quit. Find whether Kiruba is leaving RANDOM CODERS or Not!. If he is leaving print "GOODBYE KIRUBA", else print "KIRUBA IS STAYING" (without quotes).

* Note: The resultant string should be "kiruba" only, any other permutations of this word will also result in Kiruba leaving.

Input Format

* The first line contains a single integer, t representing the no. of test cases.
* The first line of each test case contains a single integer, N representing the length of string S.
* The second line of each test case contains a string, S (with lowercase English alphabets) of length N.

Constraints

* 1 <= t <= 1000
* 6 <= N <= 100

Output Format

* For every testcase, print "GOODBYE KIRUBA" if Kiruba is leaving, else print "KIRUBA IS STAYING" if Kiruba is staying.

Sample Input 0

3
6
kiruba
6
kirbua
12
zkkiarjubssa

Sample Output 0

KIRUBA IS STAYING
GOODBYE KIRUBA
KIRUBA IS STAYING

Explanation 0

* Case 1: The given word itself is kiruba thus print "KIRUBA IS STAYING"
* Case 2: There is no way to form the word kiruba after performing the given operations. Thus print "GOODBYE KIRUBA"
* Case 3: zk ki a r j ub ss a . Thus print "KIRUBA IS STAYING"

Approach

* For a given string S, first check whether the string has all the characters that the word "kiruba" has. If yes, proceed to next step, else print "GOODBYE KIRUBA".
* Then, check whether the string has all the characters in designated order. If yes, print "KIRUBA IS STAYING", else print "GOODBYE KIRUBA".
* Note 1: It is not mandatory to perform the first step. Performing only the second step will also result in correct answer.
* Note 2: Here designated order means, there is 'i' after 'k' and 'r' after 'i' and 'k' and so on in the word "kiruba". Therefore, in the given string also, make sure that 'i' is after 'k' and 'r' is after 'i' and 'k' and so on. There can be any no. of other characters in between the two required characters, but they must be in the designated order.

Code in C++

#include <iostream>

using namespace std;

int main() {
    int t;
    cin >> t;
    while(t--) {
        int n;
        cin >> n;
        string s;
        cin >> s;
        bool ans = false;
        int ch = 1;
        for(int i = 0; i < n; i++) {
            if(s[i] == 'k' && ch == 1)
                ch++;
            if(s[i] == 'i' && ch == 2)
                ch++;
            if(s[i] == 'r' && ch == 3)
                ch++;
            if(s[i] == 'u' && ch == 4)
                ch++;
            if(s[i] == 'b' && ch == 5)
                ch++;
            if(s[i] == 'a' && ch == 6) {
                ans = true;
                break;
            }
        }
        if(ans)
            cout << "KIRUBA IS STAYING" << endl;
        else
            cout << "GOODBYE KIRUBA" << endl;
    }
    return 0;
}
