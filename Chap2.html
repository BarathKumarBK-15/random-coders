Problem 1: Hands on Clock!

Problem Statement

You are given a clock that follows the twelve hour format.

Given time from 00:00 to 23:00, find the first time at which the hour hand and the minute hand overlap.

Input Format

* First line contains a single integer t, representing the number of test cases.
* The next t lines, contain a single integer time, representing time from 0 hours to 23 hours

Constraints

* 1 <= t <= 24
* 0 <= time <= 23

Output Format

* For each test case print a single line representing the time of overlap in the format hours:minutes
* Hours must be in the range 1 to 12
* Minutes must be in the range 0 to 59
* Note: In case, time is 09:03 print 9:3. Similarly if time is 12:00 print 12:0

Sample Input 0

3
23
6
12

Sample Output 0

12:0
6:32
12:0

Explanation 0

* Case 1: 23 hours represent 11 o clock in the 12 hours format.Thus the hand meet each other only after 60 minutes. Therefore output is 12:0
* Case 2: 6 hours represents 6 o clock in the 12 hours format. In such a situation the clock meets after 32 minutes and 30 seconds. Therefore ouput is 6:32
* Case 3: In this case the hands are initially at same same position. So no changes

Approach

Solution:

- The hour hand rotates at a rate of (1/2)degrees per minute
- The minute hand rotates at a rate of 6 degrees per minute
- Thus the relative rate of change in angle is 5.5 degrees per minute
- At any hour angle between hour hand and minute hand is:
angle=(hour%12)*30 (degrees)
- Thus amount of time taken to overlap in minutes is:
min=floor(angle/5.5)
- Thus the time can be calculated as (initial time % 12) + min

Code in Python

t=int(input())
for k in range(t):
    n=int(input())
    
    #Since angle between 12(minute hand) and any hour increase in steps of 30
    angle=(n%12)*30
    
    #Minute hand rotates 6 degrees and hour hand does 0.5 degrees per minutes
    #Therefore relative rotation is  6 - 0.5 = 5.5 degrees per minute
    mh=int(angle/5.5)
    
    #Move to next hour is time taken is more than 60 mins
    n+=mh//60
    
    #Since hours must be b/w 1 and 12 and minutes must be between 0 and 59
    mh%=60
    n%=12
    if n==0:
        n=12
    print("%d:%d"%(n,mh))

Code in C++

#include <iostream>

using namespace std;

int main() {
    int t;
    cin >> t;
    while(t--) {
        int time;
        cin >> time;

        //Since angle between 12(minute hand) and any hour increase in steps of 30
        int n = time % 12;
        int angle = n * 30;
        float w = 5.5f;

        //Minute hand rotates 6 degrees and hour hand does 0.5 degrees per minutes
        //Therefore relative rotation is  6 - 0.5 = 5.5 degrees per minute
        int minutes = angle / w;

        //Since hours must be b/w 1 and 12 and minutes must be between 0 and 59
        if(minutes == 60) {
            time++;
            minutes = 0;
        }
        time%=12;
        if(time == 0)
            time = 12;
        cout << time << ":" << minutes << endl;
    }
    return 0;
}

Problem 2: Growth and Fall

Problem Statement

Rachit plants a seed of a tree and expects it to reach a particular height h.The tree grows by a height, g units every week. Kiruba a naughty kid cuts the tree by f units by the end of each week.

Find the week on which the tree attains the height, h

Input Format

* The first line contains a single integer t, representing the numebr of test cases
* The next t lines contains three space seperated integers in the format "h g f", where,h is height to be reached, g is growth and f is the decrease in height

Constraints

* 1 <= t <= 1000
* 1 <= h <= 10^8
* 1 <= g <= 10^4
* 0 <= f <= 10^2
* f < g (always)

Output Format

* For each test case print a single integer representing the week on which the tree attains the height, h

Sample Input 0

2
30 4 2
45 3 0

Sample Output 0

14
15

Explanation

* Case 1: On the first week the tree grows 4 units and Kiruba cuts it by 2 units, therefore height becomes 2 units. On the second week the tree grows to a heigth of (4+2)=6 units and Kiruba cuts it to 4 units. Thus the tree will reach a height of 26 units after 13 weeks and on the 14th week the tree grows to a height of 30 units (before Kiruba cuts it)
* Case 2: The tree grows 3 units each week but Kiruba does not cuts it. Thus it reaches a height of 45 units in 15 weeks

Approach

Solution:

- By brute force, we found the resultant height at the end of each week
- The number of weeks taken to attain the height before the last growth is calculated by the formulae:
number of weeks before last growth (w) = ceil((h-g)/(g-f))
- Therefore on (w+1)th week the tree will first reach the height h

Code in Python

from math import ceil

t=int(input())
for k in range(t):
    h,g,f=list(map(int,input().split()))
    #Answer is the number of week taken to grow the previous end height + 1
    ans=ceil((h-g)/(g-f))+1
    print(ans)

Code in C++

#include <iostream>

using namespace std;

int main() {
    int t;
    cin >> t;
    while(t--) {
        int target , growth , fall;
        cin >> target >> growth >> fall;
        int ans = 0;

        //Answer is the number of week taken to grow the previous end height + 1
        ans = ((target - growth) % (growth - fall) == 0 ? (target - growth) / (growth - fall) : ((target - growth) / (growth - fall) + 1)) + 1;
        cout << ans << endl;
    }
    return 0;
}

Problem 3: Kiruba's Favourite Number

Problem Statement

Kiruba finds a set of numbers to be his favourite if they satisfy the following condition

* Any given number is represented in binary, example binary(10) => 1010
* When the binary representation is partitioned into two equal halves,then the two halves must be equal. Example 1010 => 10 | 10 .Thus 10 is one of his favourite number

Given an array A of length N. Find the number of favourite numbers in the given array.

Input Format

* The first line contains a single integer,t representing the number of test cases
* Each test case contains two lines
* The first line of each test case contains a single integer,N representing the length of the array
* The second line of each test case contains N space seperated integers

Constraints

* 1 <= t <= 100
* 1 <= N <= 45
* 1 <= A[i] <= 1000

Output Format

* For each test case print a single integer that represents the number of Kiruba's Favourite numbers in the array

Sample Input 0

2
4
1 5 15 20
1
6

Sample Output 0

2
0

Explanation 0

* Case 1:
* binary(1) => 1 => (01) => 0 | 1 .Therefore its not a favourite number.
* binary(5) => 101 => (0101) => 01 | 01 .Therefore its a favourite number.
* binary(15) => 1111 => (1111) => 11 | 11 .Therefore its a favourite number.
* binary(20) => 10100 => (010100) => 010 | 100 .Therefore its not a favourite number.
* Therefore there are 2 favourite number in the array.

* Case 2: There are no such favourite numbers in the array

Approach

Solution:

- Convert each and every number in the given array to their binary representation.
- If the length of binary string is even, divide it into two equal halves.
- If the length of binary string is odd, prepend a zero at the beginning of the binary string and divide it into halves.
- If both the halves of the binary string are equal. Then add that to the count of favourite numbers in the given array.

Code in Python

#Function to convert decimal number to binary
def dtob(dec):
    bi=""
    while dec>0:
        bi=bi+str(dec%2)
        dec//=2
    return bi[::-1]

t=int(input())
for k in range(t):
    n=int(input())
    arr=list(map(int,input().split()))
    count=0
    for i in arr:
    
    	#Convert to binary
        c=dtob(i)
        
        #If Odd add 0 in the beginning to split equally
        if(len(c)%2!=0):
            c = "0" + c
            
        #Splitting into two halves
        l=len(c)//2
        a=c[:l]
        b=c[l:]
        if(a==b):
            count+=1
    print(count)

Code in C++

#include <iostream>
#include <vector>

using namespace std;

int main () {
    int t;
    cin >> t;
    while(t--) {
        int n;
        cin >> n;
        int ans = 0;
        for(int i = 0; i < n; i++) {
            int element;
            vector <int>a;
            cin >> element;
            int length = 0;

            //Converting the decimal number to binary in reverse order of actual binary representation.
            while(element > 0) {
                length++;
                a.push_back(element % 2);
                element = element / 2;
            }

            //If length is odd in binary representation, append a zero.
            if(length % 2 != 0) {
                a.push_back(0);
                length++;
            }

            bool flag = true;

            //Checking whether the binary representation is same when split into two halves.
            for(int j = 0 , k = length / 2; j < length / 2; j++ , k++)
                if(a[j] != a[k])
                    flag = false;
            if(flag)
                ans++;
        }
        cout << ans << endl;
    }
    return 0;
}

Problem 4: Rank of a Word

Problem Statement

Given, a set of distinct lower case English alphabets, all sorted, various words are formed by permutating the given set of alphabets( each word must contain all the alphabets given ) and are arranged in lexicographic order. These words are ranked starting from 1.

Find the word for a given rank. If the given rank is greater than the number of available number of words, print -1

Input Format

* The first line contains a single integer t,representing the number of test cases
* First line of each test case contains a single integer N,representing the number of alphabets
* The second line ot each test case contains N space seperated lower case English alphabets
* The third and the last line of each test case contains a single integer R, representing the rank of the word to be found.

Constraints

* 1 <= t <= 100
* 1 <= N <= 10
* 1 <= R <= 5 x 10^6
* Each character given is always a lower case English alphabet

Output Format

* For each test case print the word in the particular rank
* If no word exist for a particular rank, print '-1'

Sample Input 0

2
5
a c d f g
48
3
x y z
8

Sample Output 0

cgfda
-1

Explanation

* Case 1: The words formed by permutating letters 'a' 'c' 'd' 'f' and 'g' are ('acdfg', 'acdgf', 'acfdg', 'acfgd', .... , 'gfdac', 'gfdca'). In the following set of words, the rank 48 is taken by the word 'cgfda'.
* Case 2: The total number of words formed by permutating 'x' 'y' and 'z' are 6. Therefore no word exist at rank 8.

Approach

Solution

- If the given rank is greater than the maximum number of permutations print -1
- Find the number of permutation by fixing each alphabet in the given set at the first position
- When the rank matches a given range of permutations, repeat the process fixing the remaining alphabets in the successive positions

Code in Python

def findrank(r,c):
    if len(c)==1:
        return c[0]
    for i in c:
        g=fact(len(c)-1)
        
        #To check if the word is in the permutated range
        if g>=r:
            x=c
            x.remove(i)
            return i+findrank(r,x)
        r-=g

def fact(f):
    ans=1
    while f>0:
        ans*=f;
        f-=1
    return ans

t=int(input())
for k in range(t):
    n=int(input())
    char=list(input().split())
    rank=int(input())
    if rank>fact(len(char)):
        print(-1)
    else:
        ans=findrank(rank,char)
        print(ans)

Code in C++

#include <iostream>
#include <vector>

using namespace std;

//To find the factorial of a given number.
int factorial(int n) {
    int fact = 1;
    for(int i = 1; i <= n; i++)
        fact = fact * i;
    return fact;
}

//To left shift the vector from a given index
void shifting(vector <char>&a , int index , int &length) {
    for(int i = index; i < length - 1; i++)
        a[i] = a[i + 1];
    length--;
}

int main() {
    int t;
    cin >> t;
    while(t--) {
        int n;
        cin >> n;
        string str("");
        vector <char>a;
        int length = n;
        for(int i = 0; i < n; i++) {
            char element;
            cin >> element;
            a.push_back(element);
        }
        int rank;
        cin >> rank;

        //If rank is greater than the max factorial value, then there is no possible solution.
        if(rank > factorial(n))
            cout << "-1" << endl;
        else {
            while(n > 0) {

                //To find the exact position of a character from the remaining set of characters in the vector that is the to be appended in the solution string.
                int r = (rank - 1) / factorial(n - 1);

                //Appending the required character in the solution string.
                str = str + a[r];

                //After appending that string should be removed from the vector, so left shifting is done.
                shifting(a , r , length);

                //Decreasing the rank according the remaining possibilities.
                rank = rank - r * factorial(n - 1);
                n--;
            }
            cout << str << endl;
        }
    }
    return 0;
}
