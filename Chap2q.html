Problem 1: Hands on Clock!

Problem Statement

You are given a clock that follows the twelve hour format.

Given time from 00:00 to 23:00, find the first time at which the hour hand and the minute hand overlap.

Input Format

* First line contains a single integer t, representing the number of test cases.
* The next t lines, contain a single integer time, representing time from 0 hours to 23 hours

Constraints

* 1 <= t <= 24
* 0 <= time <= 23

Output Format

* For each test case print a single line representing the time of overlap in the format hours:minutes
* Hours must be in the range 1 to 12
* Minutes must be in the range 0 to 59
* Note: In case, time is 09:03 print 9:3. Similarly if time is 12:00 print 12:0

Sample Input 0

3
23
6
12

Sample Output 0

12:0
6:32
12:0

Explanation 0

* Case 1: 23 hours represent 11 o clock in the 12 hours format.Thus the hand meet each other only after 60 minutes. Therefore output is 12:0
* Case 2: 6 hours represents 6 o clock in the 12 hours format. In such a situation the clock meets after 32 minutes and 30 seconds. Therefore ouput is 6:32
* Case 3: In this case the hands are initially at same same position. So no changes

Problem 2: Growth and Fall

Problem Statement

Rachit plants a seed of a tree and expects it to reach a particular height h.The tree grows by a height, g units every week. Kiruba a naughty kid cuts the tree by f units by the end of each week.

Find the week on which the tree attains the height, h

Input Format

* The first line contains a single integer t, representing the numebr of test cases
* The next t lines contains three space seperated integers in the format "h g f", where,h is height to be reached, g is growth and f is the decrease in height

Constraints

* 1 <= t <= 1000
* 1 <= h <= 10^8
* 1 <= g <= 10^4
* 0 <= f <= 10^2
* f < g (always)

Output Format

* For each test case print a single integer representing the week on which the tree attains the height, h

Sample Input 0

2
30 4 2
45 3 0

Sample Output 0

14
15

Explanation

* Case 1: On the first week the tree grows 4 units and Kiruba cuts it by 2 units, therefore height becomes 2 units. On the second week the tree grows to a heigth of (4+2)=6 units and Kiruba cuts it to 4 units. Thus the tree will reach a height of 26 units after 13 weeks and on the 14th week the tree grows to a height of 30 units (before Kiruba cuts it)
* Case 2: The tree grows 3 units each week but Kiruba does not cuts it. Thus it reaches a height of 45 units in 15 weeks

Problem 3: Kiruba's Favourite Number

Problem Statement

Kiruba finds a set of numbers to be his favourite if they satisfy the following condition

* Any given number is represented in binary, example binary(10) => 1010
* When the binary representation is partitioned into two equal halves,then the two halves must be equal. Example 1010 => 10 | 10 .Thus 10 is one of his favourite number

Given an array A of length N. Find the number of favourite numbers in the given array.

Input Format

* The first line contains a single integer,t representing the number of test cases
* Each test case contains two lines
* The first line of each test case contains a single integer,N representing the length of the array
* The second line of each test case contains N space seperated integers

Constraints

* 1 <= t <= 100
* 1 <= N <= 45
* 1 <= A[i] <= 1000

Output Format

* For each test case print a single integer that represents the number of Kiruba's Favourite numbers in the array

Sample Input 0

2
4
1 5 15 20
1
6

Sample Output 0

2
0

Explanation 0

* Case 1:
* binary(1) => 1 => (01) => 0 | 1 .Therefore its not a favourite number.
* binary(5) => 101 => (0101) => 01 | 01 .Therefore its a favourite number.
* binary(15) => 1111 => (1111) => 11 | 11 .Therefore its a favourite number.
* binary(20) => 10100 => (010100) => 010 | 100 .Therefore its not a favourite number.
* Therefore there are 2 favourite number in the array.

* Case 2: There are no such favourite numbers in the array

Problem 4: Rank of a Word

Problem Statement

Given, a set of distinct lower case English alphabets, all sorted, various words are formed by permutating the given set of alphabets( each word must contain all the alphabets given ) and are arranged in lexicographic order. These words are ranked starting from 1.

Find the word for a given rank. If the given rank is greater than the number of available number of words, print -1

Input Format

* The first line contains a single integer t,representing the number of test cases
* First line of each test case contains a single integer N,representing the number of alphabets
* The second line ot each test case contains N space seperated lower case English alphabets
* The third and the last line of each test case contains a single integer R, representing the rank of the word to be found.

Constraints

* 1 <= t <= 100
* 1 <= N <= 10
* 1 <= R <= 5 x 10^6
* Each character given is always a lower case English alphabet

Output Format

* For each test case print the word in the particular rank
* If no word exist for a particular rank, print '-1'

Sample Input 0

2
5
a c d f g
48
3
x y z
8

Sample Output 0

cgfda
-1

Explanation

* Case 1: The words formed by permutating letters 'a' 'c' 'd' 'f' and 'g' are ('acdfg', 'acdgf', 'acfdg', 'acfgd', .... , 'gfdac', 'gfdca'). In the following set of words, the rank 48 is taken by the word 'cgfda'.
* Case 2: The total number of words formed by permutating 'x' 'y' and 'z' are 6. Therefore no word exist at rank 8.
